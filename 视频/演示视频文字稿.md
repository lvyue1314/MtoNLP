# ChartQA-X 三模型评测系统 — 演示视频文字稿

> 时长: 2 分 30 秒 | 建议录制时打开 Gradio 界面 + 切换 PPT 关键图表

---

## 第 1 段：项目介绍（20 秒）

【画面】PPT 第 1 页：项目标题 + 三类模型对比表

```
大家好，我们做的是一个论文图表问答评测系统。

核心目标是对比三种多模态技术路线在图表理解上的表现：
rule基线——不看图，只从问题文本猜答案
LLaVA——外挂式多模态，通过ViT+投影层把图像"翻译"给语言模型
Gemma 4——原生多模态，图像直接从输入层就进入文本空间

我们想看：不同的融合方式，到底差在哪里？
```

---

## 第 2 段：Gradio 系统演示（40 秒）

【画面】打开 Gradio 公网链接，展示界面 + 上传 chart_000001.png

```
这就是我们的演示系统，用 Gradio 搭建，可以通过公网链接访问。

上传一张图表——这是 1850-2015 年食品价格指数的折线图，
包含 Lamb、Corn、Beef 等十几条曲线。

我们问它一个问题："What is the difference in value between Lamb and Corn?"

勾选 Gemma 4 模型，点击提交。
```

【画面】等待 3 秒，Gemma 4 返回结果并高亮 103.7 - 103.13 = 0.57

```
Gemma 4 准确地找到了 Lamb 和 Corn 两条曲线对应的数值，
计算出精确差值 0.57。完全正确。
```

---

## 第 3 段：成功案例 vs 失败案例（30 秒）

【画面】切换到 LLaVA 的回答

```
再来看看 LLaVA 对同一张图的回答。
它知道题目问的是 Lamb 和 Corn，但它说"lamb is protein-rich，
corn is a staple grain"——它在解释为什么价格不同，
却说不清具体差值是多少。

这就是外挂式多模态的典型缺陷：
ViT→Projector 管道在传输精确数值时丢失了像素级精度。
它可以理解图表的语义，但读不准具体的数字。
```

---

## 第 4 段：评测结果展示（30 秒）

【画面】切换到评测结果对比表 + 柱状图

```
我们在 ChatQA-X 数据集上对三个模型做了 2500 道题的批量评测。
结果非常清晰：

Gemma 4 的 Contains Match 是 74.6%，
LLaVA 只有 11.2%，
规则基线更只有 6.2%。

这个 6.7 倍的差距，就是原生融合 vs 外挂式融合的根本差异。
```

---

## 第 5 段：t-SNE 交融机制可视化（20 秒）

【画面】切换到 t-SNE 矩阵图 + 跨模态相似度曲线

```
我们通过 t-SNE 降维可视化，深入分析了两个多模态模型
内部视觉Token和文本Token的融合过程。

这张图非常直观：
左边 Gemma 4，从第 0 层开始红色和蓝色就混在一起——visual 和 text 早就融合了。
右边 LLaVA，前 24 层红色和蓝色完全分离——视觉和文本在"各说各话"，
要等到深层才勉强靠近。

相似度曲线更清楚：
Gemma 4 起步就是 0.73，第 4 层就启动了融合。
LLaVA 起步只有 0.11，要花 24 层来"翻译"两个不同空间。

这就是为什么 Gemma 4 能读准数字、LLaVA 读不准——
它们花在"理解图表"上的有效层数差了 3 倍。
```
