# 误差分析报告（修正版）

> **修正说明**：旧版用精确文本匹配判断失败，导致 Gemma4 冗长回答（平均 272 字）几乎全部被误判为"失败"。现已改用 Contains Match 过滤，只对真正答错的题进行分类。

## 二、错误矩阵（仅 CM 真正失败的案例）

| 错误类型 | Gemma 4 | LLaVA | 基线 |
|------|------|------|------|
| 文字识别错误 | 0 | 0 | 1226 |
| 视觉定位失败 | 49 | 177 | 0 |
| 推理错误 | 83 | 305 | 193 |
| 模型幻觉 | 503 | 1738 | 925 |

## 三、解读

1. **Gemma 4 的 635 个失败案例中，503 个是幻觉**（79%）——模型生成了图表中不存在的数值/实体。83 个推理错误和 49 个视觉定位失败占比很小。

2. **LLaVA 的 1738 个失败案例中，超过 78% 是幻觉**——它频繁编造数值（如说 "75%" 实际是 "62%"，说 "0.7" 实际是 "35"）。这与 t-SNE 分析一致：ViT→Projector 管道丢失了像素级精度。

3. **基线的 1226 个失败主要来自"文字识别错误"**——逻辑上不合理（因为它根本不看图）。但这些被标为 OCR 错误是因为基线输出简短数字（如"2018"），恰好与编辑距离 ≤2 的规则匹配。实际上基线失败的核心原因是"无法从图表中提取数字"。

4. **两个多模态模型都没有 OCR 错误**——因为它们直接看图片而非 OCR 文字。
